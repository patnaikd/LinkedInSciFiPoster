Three approaches to the new workflow

Image generation in Authoring + manual copy-paste publishing

A

Side-by-side panels

Authoring page splits into two columns: left = post text editor (existing), right = image generation panel with fal.ai prompt + preview + download button. Publishing page becomes a simple "How to post" checklist with no LinkedIn connection UI.

Pros

  • Everything visible at once
  • Natural left→right flow

Cons

  • Crowded on smaller screens
  • Existing sidebar already competes for space
B

Stacked sections (recommended)

Authoring page keeps existing layout. A new "Image" section appears below the text editor — fal.ai prompt field, generate button, preview, download. Publishing page replaces LinkedIn OAuth with a step-by-step manual posting guide (copy text button + download image link).

Pros

  • Minimal disruption to existing layout
  • Works on all screen sizes
  • Least risky change

Cons

  • Requires scrolling to reach image section
C

Tabs within Authoring

Authoring page gains two tabs: "Post Text" (current editor) and "Image" (fal.ai generator). Publishing page becomes the manual guide. User switches between tabs as they work.

Pros

  • Clean separation, no scroll
  • Good on all screen sizes

Cons

  • Can't see text while generating image
  • Slightly more complex nav
